The One Thing to Be Aware Of
The seed functions are designed to be idempotent — they check "does data already exist?" before inserting. For example, seedEsopGrants checks if (existing.length > 0) return. This means:

If you manually delete a grant through the UI, the seed function will recreate it on next restart
If you manually add grants, the seed function won't touch them (it sees existing data and skips)
If I add a new seed function for a new instrument, it will only insert if that table is empty for the tenant
This is the correct behavior for a development workbench. Your manual changes persist, and new seed data only fills in gaps.
